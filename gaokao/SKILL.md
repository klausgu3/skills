# 高考志愿规划（可复用）

帮考生从 **约束定稿 → 专业收敛 → 院校池 → 冲稳保表 → 调剂风控 → 填报 checklist** 产出成套 Markdown 文档。风格：**本科好就业、少考研考公、直白可执行**（类似张雪峰）。

**硬性要求：完整走本 Skill 时，必须把结果写入工作区 `.md` 文件，不能只写在聊天里。** 聊天回复可摘要 + 指向文件路径；主交付物是磁盘上的 Markdown。

## 何时用 / 不用

| 场景 | 处理 |
|------|------|
| 给新分数/位次做志愿、改约束、加列（院校代码/备注） | ✅ 走本 Skill，**并 Write `.md` 到工作区** |
| 只问某个专业是什么、某校好不好（单次问答） | ❌ 直接答，不必全套文档 |
| 用户未说改代码且只问政策 | ❌ 只答问题 |

## 工作流总览

```
Phase 0 信息采集 → Phase 1 约束定稿 → Phase 2 专业清单
→ Phase 3 院校池 → Phase 4 位次对标与分档 → Phase 5 产出文档
→ Phase 6 风控与核对
```

复制 checklist 并逐项打勾：

```
- [ ] Phase 0：分数/位次/选科/省份/批次线已确认
- [ ] Phase 1：约束表已定稿（地域/就业/排除项）
- [ ] Phase 2：专业 ABC 线与组内禁填已写清
- [ ] Phase 3：院校池符合约束（无师范/民办等若用户要求）
- [ ] Phase 4：冲稳保位次区间与标记 🟢🟡🔴 已标
- [ ] Phase 5：**已写入工作区 `.md` 文件**（约束 md + 志愿表 md）；志愿表含 院校代码+组+备注 列
- [ ] Phase 6：调剂风险表 + checklist 已完成
```

## Phase 0：信息采集（缺一项先问）

**必填**

| 字段 | 示例 |
|------|------|
| 省份 | 江西 |
| 高考年份 | 2026 |
| 科类/选科 | 物理类 · 物+化+地 |
| 分数 + 位次 | 508 / 80283 |
| 当年特控线、本科线（分+位次） | 505/83293 · 412/173871 |

**就业与偏好（决定专业排序）**

| 维度 | 常见选项 |
|------|----------|
| 就业目标 | 本科毕业好就业；是否考研/考公 |
| 工作形态 | 进厂设计/设备/维修（非产线）、出差、倒班、保密单位 |
| 考试偏好 | 如「不想考试」→ 电气 **降权**，不以电网为主攻 |
| 专业主线 | 默认 **B＞A＞C**（见下） |
| 地域 | 省内 + 周边省列表；**不去** 的片区 |
| 院校性质 | 公办/是否接受民办/中外合作/师范 |

**若用户只说分数不说位次**：提醒查当年一分一段表；可 WebSearch「{省} {年} 一分一段表 {科类} {分}」。

## Phase 1：约束定稿

产出 **`志愿整理-主线{标签}.md`**（标签可用 ABC 或考生简称）。

### 1.1 约束表模板

| 维度 | 内容 |
|------|------|
| 地域 | 可报 / 排除（写清省或城市） |
| 就业目标 | … |
| 工作形态 | … |
| 主线 | A 计算机/安全 · B 电信/自动化 · C 汽车/机电/电气(备选) |
| 考试/体制 | 电网、公务员等是否排除 |
| 院校 | 公办/民办/师范/中外合作 |

### 1.2 整类删除 vs 组内禁填

- **整校不报**：校名含师范大学/师范学院/科技师范大学；独立学院；民办；中外合作（按用户要求）
- **组内勿填**：小学教育、师范类、医学、法学、金融、市场营销、纯化学生物等（防调剂）
- **整组不报**：组内混杂经管+工科且无法只填工科时标 🔴

记录 **已删除方向** 及原因（避免后续又加回）。

## Phase 2：专业清单

### 默认 ABC 线（可按考生改优先级）

| 线 | 代表专业 | 定位 |
|----|----------|------|
| **A** | 计算机、软件、信息安全 | 高薪上限，卷；校一般时作次选 |
| **B** | 电子信息、自动化、机器人、机电 | **最贴设备/硬件/进厂研发**，本科就业稳 |
| **C** | 新能源汽车、智能制造、机设；电气末位 | 装备/车企链；电气非电网 |

### 组内专业顺序（默认 B＞A＞C）

```
1. 电信 / 电子信息类
2. 自动化 / 机器人 / 机电
3. 计算机 / 软件
4. 机设 / 智能制造 / 新能源
5. 电气（末位，标注非电网）
```

每组 **只填 2～4 个** 目标专业，不留空位给不喜欢的专业。

## Phase 3：院校池

### 3.1 筛选顺序

1. **地域**：只在可报省份选
2. **层次**：公办本科（大学/学院），按用户排除民办等
3. **校名**：排除师范类校名（若用户要求）
4. **专业组**：该校在目标省有 **工科组** 且含 B/A 线专业
5. **中外/专项**：单独标注，默认不填

### 3.2 江西 + 鄂湘皖闽 公办池（参考）

详见 [reference-jx.md](reference-jx.md)。新考生位次不同时 **重新查近 2～3 年组投档位次**，不要照搬旧表。

### 3.3 院校代码

- 填报用的是 **本省四位院校代号**（江西示例：井冈山 8216），不是教育部 5 位国标码
- 用 WebSearch：`{年} {校名} 江西 招生代码` 或查省考试院计划
- 查不到标 **`查{年}`**，勿编造
- 正式填报以当年《高考天地》/志愿填报系统为准

## Phase 4：位次对标与冲稳保

### 4.1 分档规则（按考生位次 R）

| 档次 | 位次区间（相对 R） | 数量建议 | 策略 |
|------|-------------------|----------|------|
| **冲** | 约 0.8R～R（更高校） | 8～15 组 | 省外工科、贴线组 |
| **稳** | 约 R～1.1R | 15～20 组 | **主战场**；省内 ≥3 所 |
| **保** | 约 1.1R～1.5R+ | 10～15 组 | 防滑；≥2 组位次 **＞10 万**（按省规模调整） |

用 **最近一年组投档位次** 为主，前一年校验趋势。标注：

- 🟢 组内工科纯、调剂风险低
- 🟡 需查计划 / 位次边界 / 组内混专业
- 🔴 不要报（整组作废）

### 4.2 数据获取

```
WebSearch: {校名} {省} {年} 专业组 {501/502} 投档位次
WebSearch: {校名} 江西 招生代码
```

无数据时：位次列写「查{年}」，说明中写查证路径。

### 4.3 重复与互斥

- 同一校同一组在冲/稳中 **勿重复**（保档可刻意重复防滑时注明）
- 上冲组（位次明显优于 R）可放在稳区末尾作「可冲」

## Phase 5：产出文档（必写磁盘，不可仅聊天输出）

**触发条件**：用户要做志愿规划、冲稳保、位次对标、或给出分数/位次求填报指导 → **必须**用 Write 工具（或等价方式）在工作目录写入 Markdown。**禁止**只在对话里给完整志愿表而不落盘。

**默认写入路径**：当前工作区根目录（如 `E:\Documents\myWorkspace\guzhenyu`）；用户指定目录时从其指定。

生成 **至少 2 个、建议 3～4 个** 文件：

| 文件 | 内容 | 是否必写 |
|------|------|----------|
| `志愿表-{分}-{位次简写}.md` | **主交付**：冲稳保表 + 填报顺序 + 风控 + checklist | **必写** |
| `2 志愿整理-主线{分或标签}.md` 或 `1 专业粗筛.md` | 约束 + ABC 专业清单 + 目标校评估 | **必写** |
| `0 高考志愿-专业全景分析.md` | 候选专业就业/课程/风险 | 用户已有则更新；无则按需新建 |

**位次简写**：71421 → `7.1万位次`；80283 → `8万位次`（全文一致即可）。

**写完后**：在聊天中告知用户**已生成的文件名与路径**，可附 3～5 句摘要；**不要把整份志愿表重复粘贴进聊天**（除非用户明确要求「只要聊天不要文件」）。

### 5.1 志愿表必备结构

1. 文首：筛选规则、考生信息、**院校代码说明**
2. 院校筛选原则（排除/可报/组内勿填）
3. 位次定位表（分数、特控线、本科线、策略）
4. 专业顺序（B＞A＞C）
5. **【冲】【稳】【保】表格** — 列顺序固定：

```
| 序 | 省 | 院校 | 院校代码 | 组 | {年}位次 | 标记 | 说明 | 备注 |
```

6. 系统填报顺序（文字链）
7. 时间紧 · 最少 N 组（精简版）
8. 调剂风险表
9. 近 3 年位次对照（核心校）
10. 填报 checklist

完整表头与段落模板见 [templates.md](templates.md)。

### 5.2 说明列写法

每条说明含：**组内专业优先级**（①②③）、**为何在此档**、**特殊警告**（中外/停招/经管组）。

## Phase 6：风控与核对

### 6.1 调剂风险（必做）

| 等级 | 典型情况 |
|------|----------|
| ⚠️⚠️⚠️ | 组内师范/医学/经管/化工，或校名非师范但组很杂 |
| ⚠️⚠️ | 材料/文理大组、501 混合组 |
| ⚠️ | 非师范校但组内或有师范 |
| ✅ | 502 等物+化纯工科组 |

### 6.2 填报 checklist（每考生定制勾选）

- [ ] 无用户禁止的院校类型
- [ ] 危险组（🔴）未进入正式填报顺序
- [ ] 每组 2～4 专业，无师范/医学等禁填
- [ ] 院校代码 + 组号对照当年计划
- [ ] 稳档省内数量、保档位次下限达标

---

## 与用户沟通原则

1. **用中文**，表格清晰，少空话
2. **位次优先于分数** 对标学校
3. 不承诺录取；写清数据年份与「以考试院为准」
4. 用户改约束（如删师范、加地域）→ **整表重筛并重写 md 文件**，不是只改一行
5. **完整志愿规划必须输出 `.md` 到工作区**；不编译、不修改项目代码（仅写 Markdown 文档）
6. 用户说「输出到 md / 写文档 / 更新 skill」→ 执行 Write + 必要时更新本 Skill 的 Phase 5 约定

## 快速启动（新考生一句话）

用户：「江西物化地 520 分 6 万位，不要师范，好就业」

1. 确认特控/本科线 → 2. 问清就业/地域/是否考研 → 3. **Write** `2 志愿整理-主线*.md` → 4. 查院校组位次 → 5. 分冲稳保 → 6. **Write** `志愿表-{分}-{位次简写}.md` → 7. checklist → 8. 聊天告知文件路径
